# Managing Kernel Resources

บางครั้ง PostgreSQL อาจใช้ทรัพยากรต่างๆ ของ OS จนหมด โดยเฉพาะเมื่อมีการใช้งานหลาย instances บนระบบเดียวกัน หรือ instance มีขนาดใหญ่มาก ส่วนนี้จะอธิบายถึง kernel resource ที่ PostgreSQL ใช้ และขั้นตอนที่คุณสามารถทำได้ เพื่อแก้ไขปัญหาที่เกี่ยวข้องกับการใช้ kernel resource

## Shared Memory and Semaphores

PostgreSQL ต้องการให้ OS มีคุณสมบัติการสื่อสารระหว่างกระบวนการ (IPC) โดยเฉพาะอย่างยิ่ง shared memory และ semaphores ระบบที่พัฒนามาจาก Unix โดยทั่วไปจะมี IPC แบบ System V (เก่า) หรือ POSIX (ใหม่) หรือทั้งสองแบบ

โดยค่าเริ่มต้น PostgreSQL จะจัดสรร shared memory แบบ System V จำนวนเล็กน้อย และ anonymous mmap จำนวนมาก หรืออาจใช้แบบ System V ขนาดใหญ่เพียงพื้นที่เดียวก็ได้ (shared_memory_type) <br>
นอกจากนี้ ยังมีการสร้าง semaphores จำนวนมาก ซึ่งอาจเป็นแบบ System V หรือ POSIX ในระหว่างการเริ่มต้นเซิร์ฟเวอร์ ปัจจุบัน semaphores แบบ POSIX ถูกใช้ในระบบ Linux และ FreeBSD ในขณะที่แพลตฟอร์มอื่นๆ ใช้ semaphores แบบ System V

หากต้องการดูว่าในระบบปัจจุบันมี IPC ตัวไหนที่ กำลังรันอยู่ ให้ใช้คำสั่งเหล่านี้:

```
# System V
ipcs -m

# POSIX
ls -lah /dev/shm
หรือ
ls -lah /dev/mqueue
```

คุณสมบัติ IPC ของ System V มักถูกจำกัดด้วยขีดจำกัดการจัดสรรทั่วทั้งระบบ เมื่อ PostgreSQL เกินขีดจำกัดเหล่านี้ เซิร์ฟเวอร์จะปฏิเสธการเริ่มต้น และควรทิ้งข้อความแสดงข้อผิดพลาดที่ให้คำแนะนำไว้ อธิบายเกี่ยวกับปัญหาและวิธีแก้ไข
