# The PostgreSQL User Account

เช่นเดียวกับ Server ทั่วไป ที่เปิดให้โลกภายนอกเข้าถึงได้ มีคำแนะนำที่สำคัญมากดังนี้:

- ต้องใช้ Account แยกต่างหาก: ควรตั้ง User ขึ้นมาใหม่ (ปกติคือ postgres) เพื่อรัน PostgreSQL โดยเฉพาะ
- ห้ามแชร์ User ร่วมกับบริการอื่น: อย่าใช้ User เดียวกันไปรันอย่างอื่น (เช่น ห้ามใช้ nobody เพราะ nobody มักจะถูกใช้โดยบริการจิปาถะหลายตัว ถ้าตัวหนึ่งโดนเจาะ ตัวอื่นจะพลอยซวยไปด้วย)
- สิทธิ์เหนือข้อมูลเท่านั้น: User นี้ควรมีสิทธิ์ "เป็นเจ้าของ" (Owner) เฉพาะ ไฟล์ข้อมูล (Data Files) ที่ PostgreSQL จัดการเท่านั้น

```
ubuntu:~$ sudo adduser --system --group --shell /bin/bash --home /var/lib/postgresql postgres
```

## systemd RemoveIPC

หากมีการใช้งาน `systemd` ต้องระมัดระวังในการใช้ทรัพยากร `IPC` (รวมถึง `shared memory`) ถูก `OS` ลบทิ้งก่อนเวลาอันควร โดยเฉพาะอย่างยิ่งเมื่อติดตั้ง PostgreSQL จากซอร์สโค้ด เพราะการติดตั้งแบบอื่น ผู้ใช้ `postgres` จะถูกสร้างขึ้นเป็น `system user`

การตั้งค่า RemoveIPC จะอยู่ในในไฟล์ `/etc/systemd/logind.conf` จะควบคุมว่า IPC จะถูกลบออกหรือไม่ เมื่อผู้ใช้ล็อกเอาต์อย่างสมบูรณ์ (ยกเว้น `system user`) การตั้งค่านี้มีค่าเริ่มต้นเป็น `เปิด` ใน `systemd` มาตรฐาน แต่ระบบปฏิบัติการบางรุ่นอาจมีค่าเริ่มต้นเป็นปิด

ผลกระทบที่พบได้ทั่วไป เมื่อเปิดใช้งานการตั้งค่านี้คือ `shared memory objects` ซึ่งใช้สำหรับ `Parallel Query Execution` จะถูกลบออกในเวลาที่ไม่แน่นอน ส่งผลให้เกิดข้อผิดพลาด เช่น

```
WARNING:  could not remove shared memory segment "/PostgreSQL.1450751626": No such file or directory
```
