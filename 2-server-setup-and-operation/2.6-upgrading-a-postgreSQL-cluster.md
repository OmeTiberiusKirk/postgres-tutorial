# Upgrading a PostgreSQL Cluster

หมายเลขเวอร์ชันของ PostgreSQL ในปัจจุบันประกอบด้วย `หลัก` และ `ย่อย`. ตัวอย่างเช่น, เวอร์ชั่น 10.1, เลข 10 คือหมายเลขเวอร์ชันหลัก และเลข 1 คือหมายเลขเวอร์ชันย่อย, ซึ่งหมายความว่านี่คือเวอร์ชันแรกของเวอร์ชัน 10. ในกรณี 9.5.3, เวอร์ชันหลักจะประกอบด้วยตัวเลขสองหลักแรก คือ 9.5, and the minor version is the third number, e.g., 3 และหมายเลขเวอร์ชันย่อยคือตัวเลขหลักที่สาม คือ 3, ซึ่งหมายความว่านี่คือเวอร์ชันที่ 3 ของเวอร์ชัน 9.5.

การอัปเดตเวอร์ชันย่อย จะไม่เปลี่ยนแปลงรูปแบบการจัดเก็บข้อมูลภายใน และจะเข้ากันได้กับเวอร์ชันย่อยก่อนหน้า และเวอร์ชันย่อยถัดไปของหมายเลขเวอร์ชันหลักเดียวกันเสมอ. ตัวอย่างเช่น, เวอร์ชัน 10.1 จะเข้ากันได้กับเวอร์ชัน 10.0 และเวอร์ชัน 10.6. ในทำนองเดียวกัน 9.5.3 ก็เข้ากันได้กับ 9.5.0, 9.5.1 และ 9.5.6. ในการอัปเดตระหว่างเวอร์ชันที่เข้ากันได้, เพียงแค่เปลี่ยนไฟล์ปฏิบัติการ ในขณะที่เซิร์ฟเวอร์ปิดอยู่ แล้วรีสตาร์ทเซิร์ฟเวอร์. `data directory` จะไม่เปลี่ยนแปลง — การอัปเกรดเวอร์ชันย่อยนั้นง่ายมาก.

สำหรับการอัปเดตเวอร์ชันหลักของ PostgreSQL, รูปแบบการจัดเก็บข้อมูลภายในอาจมีการเปลี่ยนแปลง, ทำให้การอัปเกรดมีความซับซ้อนมากขึ้น. วิธีการดั้งเดิมในการย้ายข้อมูลไปยังเวอร์ชันหลักใหม่คือ `dump` และ `restore`, แต่กระบวนการนี้อาจช้า. วิธีที่เร็วกว่าคือ pg_upgrade.

## Upgrading Data via pg_dumpall

คือการสำรองข้อมูลจาก PostgreSQL เวอร์ชันหลักหนึ่ง ไปอีกเวอร์ชันหนึ่ง — ในการทำเช่นนี้, ต้องใช้เครื่องมือสำรองข้อมูลเชิงตรรกะ เช่น pg_dumpall; วิธีการสำรองข้อมูลระดับระบบไฟล์จะไม่สามารถใช้งานได้.

แนะนำให้ใช้โปรแกรม pg_dump และ pg_dumpall จาก PostgreSQL เวอร์ชันใหม่, เพื่อใช้ประโยชน์จากการปรับปรุงที่อาจได้รับในโปรแกรมเหล่านี้. เวอร์ชันปัจจุบัน สามารถอ่านข้อมูลจากเซิร์ฟเวอร์ เวอร์ชันใดก็ได้ย้อนหลังไปถึง 9.2.

1. หากทำการสำรองข้อมูลโปรดตรวจสอบให้แน่ใจว่า ฐานข้อมูลไม่ได้อยู่ระหว่างการอัปเดต. สิ่งนี้จะไม่ส่งผลกระทบ ต่อความสมบูรณ์ของข้อมูล, แต่ข้อมูลที่เปลี่ยนแปลงไปจะไม่รวมอยู่ในการสำรองข้อมูล จึงจำเป็นต้องแก้ไขสิทธิ์ การเข้าถึงในไฟล์ /usr/local/pgsql-17.9/data/pg_hba.conf (หรือไฟล์ที่เทียบเท่า) เพื่อไม่อนุญาตให้บุคคลอื่นเข้าถึง.

   ```
   # /usr/local/pgsql-17.9/data/pg_hba.conf

   # TYPE DATABASE USER  ADDRESS       METHOD
   # ปฏิเสธการเชื่อมต่ออื่นๆ ทั้งหมด(หรือใส่ # ปิดบรรทัดอื่นไปให้หมด)
   host   all      all   0.0.0.0/0     reject

   # อนุญาตเฉพาะตัวเราเองที่เชื่อมต่อจากเครื่อง server ตรงๆ
   local  all      all                 trust
   host   all      all   127.0.0.1/32  trust
   ```

   ในการสำรองข้อมูล, คุณสามารถใช้คำสั่ง pg_dumpall จากเวอร์ชันที่คุณใช้งานอยู่ปัจจุบันได้. อย่างไรก็ตาม เพื่อผลลัพธ์ที่ดีที่สุด, ควรลองใช้คำสั่ง pg_dumpall จาก PostgreSQL 18.3, เนื่องจากเวอร์ชันนี้ มีการแก้ไขข้อบกพร่อง และการปรับปรุงที่ดีกว่าเวอร์ชันเก่า.

2. ทำการเริ่ม server ใหม่โดยใช้คำสั่ง:

   ```
   postgres:~$ pg_ctl -l $PGDATA/logfile restart
   ```

3. ในการสำรองข้อมูล, ให้พิมพ์:

   ```
   postgres:~$ /usr/local/pgsql-18.3/bin/pg_dumpall > ~/dumpfile
   ```

4. Shut down เซิร์ฟเวอร์เก่า:

   ```
   postgres:~$ pg_ctl -l $PGDATA/logfile stop
   # หรือ
   ubuntu:~$ sudo systemctl stop postgresql
   ```

5. กำหนดค่าโครงสร้างซอร์สโค้ดให้กับ PostgreSQL เวอร์ชันใหม่ จากนั้นทำการติดตั้ง

   ```
   ubuntu:~/postgresql-18.3$ ./configure \
   --with-systemd \
   --with-lz4 \
   --with-liburing \
   --with-openssl \
   --with-icu \
   --prefix=/usr/local/pgsql-18.3 \
   LDFLAGS="-Wl,-rpath,'/usr/local/pgsql-18.3/lib'"

   ubuntu:~/postgresql-18.3$ make world
   ubuntu:~/postgresql-18.3$ make install-world
   ```

6. สร้าง `database cluster` ใหม่:

   ```
   postgres:~$ /usr/local/pgsql-18.3/bin/initdb \
   -D /usr/local/pgsql-18.3/data \
   --locale-provider=icu \
   --icu-locale=th-TH \
   --encoding=UTF8
   ```

7. คืนค่าไฟล์ pg_hba.conf และการแก้ไขใดๆ ในไฟล์ postgresql.conf.
8. เริ่มต้น database server, อีกครั้ง:

   ```
   postgres:~$ /usr/local/pgsql-18.3/bin/pg_ctl \
   -D /usr/local/pgsql-18.3/data \
   -l /usr/local/pgsql-18.3/data/logfile start
   ```

9. สุดท้ายนี้, ให้กู้คืนข้อมูลจากไฟล์สำรองโดยใช้คำสั่ง:

   ```
   postgres:~$ /usr/local/pgsql-18.3/bin/psql -f ~/dumpfile
   ```

   โดยใช้ psql เวอร์ชั่นใหม่.

สามารถลดเวลาหยุดทำงานให้น้อยที่สุดได้ โดยการติดตั้ง PostgreSQL ใหม่ใน directory ที่ต่างกัน และทำงานทั้งเก่าและใหม่พร้อมกัน, บน port ที่ต่างกัน. จากนั้นใช้คำสั่งดังนี้:

```
postgres:~$ /usr/local/pgsql-18.3/bin/pg_dumpall -p 5432 | \
/usr/local/pgsql-18.3/bin/psql -p 5433
```

## Upgrading Data via pg_upgrade

```
postgres:~$ pg_checksums --enable

postgres:~$ /usr/local/pgsql-18.3/bin/pg_upgrade \
--old-datadir /usr/local/pgsql-17.9/data \
--new-datadir /usr/local/pgsql-18.3/data \
--old-bindir /usr/local/pgsql-17.9/bin \
--new-bindir /usr/local/pgsql-18.3/bin \
--check

postgres:~$ /usr/local/pgsql-18.3/bin/pg_upgrade \
--old-datadir /usr/local/pgsql-17.9/data \
--new-datadir /usr/local/pgsql-18.3/data \
--old-bindir /usr/local/pgsql-17.9/bin \
--new-bindir /usr/local/pgsql-18.3/bin
```
