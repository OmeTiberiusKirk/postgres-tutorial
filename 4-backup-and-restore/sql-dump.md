# SQL Dump
มี 3 แนวทางพื้นฐาน ในการสำรองข้อมูล PostgreSQL:
- SQL dump
- File system level backup
- Continuous archiving

แต่ละวิธีมีจุดแข็งและจุดอ่อนของตนเอง ซึ่งจะกล่าวถึงในรายละเอียดของแต่หละวิธี ในหัวข้อต่อไปนี้

แนวคิดเบื้องหลังวิธีการดัมพ์นี้ คือการสร้างไฟล์ที่มีคำสั่ง SQL, เมื่อป้อนกลับไปยังเซิร์ฟเวอร์แล้ว, จะสร้างฐานข้อมูลขึ้นใหม่ ในสถานะเดียวกับที่เป็นอยู่ขณะที่ทำการดัมพ์ข้อมูล. 

```
pg_dump dbname > dumpfile
```

ในขณะที่คำสั่งข้างต้นสร้างไฟล์ข้อความ, สามารถสร้างไฟล์ในรูปแบบอื่นๆ ที่ช่วยให้สามารถประมวลผลแบบขนาน ควบคุมการกู้คืนวัตถุได้อย่างละเอียดมากขึ้น.

pg_dump เป็นโปรแกรมไคลเอ็นต์ PostgreSQL ทั่วไป (ถึงแม้จะเป็นโปรแกรมที่ฉลาดเป็นพิเศษก็ตาม). นั่นหมายความว่า สามารถทำการสำรองข้อมูลจากโฮสต์ระยะไกล ที่สามารถเข้าถึงฐานข้อมูลได้. แต่โปรดจำไว้ว่า pg_dump ไม่ทำงานด้วยสิทธิ์พิเศษ. โดยเฉพาะอย่างยิ่ง, มันต้องมีสิทธิ์ในการอ่านตารางทั้งหมด ที่ต้องการสำรองข้อมูล, ดังนั้นในการสำรองข้อมูลฐานข้อมูลทั้งหมด เกือบจะต้องเรียกใช้มันในฐานะผู้ใช้ระดับสูงสุดของฐานข้อมูล. (หากไม่มีสิทธิ์เพียงพอที่จะสำรองข้อมูลฐานข้อมูลทั้งหมด, คุณยังสามารถสำรองข้อมูลบางส่วนของฐานข้อมูล ที่คุณสามารถเข้าถึงได้โดยใช้ตัวเลือกต่างๆ เช่น -n schema หรือ -t table.)

ข้อดีที่สำคัญอย่างหนึ่งของ pg_dump ข้อมูลที่ได้จาก pg_dump สามารถนำไป import เข้าไปใน PostgreSQL เวอร์ชันใหม่กว่าได้, แต่ทว่า file-level backups และ continuous archiving นั้นขึ้นอยู่กับเวอร์ชันของเซิร์ฟเวอร์. 

pg_dump เป็นวิธีเดียวที่สามารถถ่ายโอนฐานข้อมูล ไปยังเครื่องที่มี architecture ที่แตกต่างกันได้ เช่น จากเซิร์ฟเวอร์ 32 บิตไปยังเซิร์ฟเวอร์ 64 บิต

## Restoring the Dump
```
psql -X test < dumpfile
```

คำสั่งนี้จะไม่สร้างฐานข้อมูล test ดังนั้นคุณต้องสร้างฐานข้อมูลด้วยตนเอง (เช่น createdb test). 

โดยค่าเริ่มต้น สคริปต์ psql จะยังคงทำงานต่อไปแม้ว่าจะพบข้อผิดพลาด SQL ก็ตาม. อาจต้องการเรียกใช้ psql โดยตั้งค่าตัวแปร ON_ERROR_STOP เพื่อเปลี่ยนแปลงพฤติกรรมนั้น และให้ psql ออก หากเกิดข้อผิดพลาด SQL:

```
psql -X --set ON_ERROR_STOP=on dbname < dumpfile
```

หรืออีกทางเลือกหนึ่ง, สามารถระบุให้กู้คืนข้อมูลทั้งหมดเป็น single transaction, เพื่อให้การกู้คืนเสร็จสมบูรณ์ หรือถูกยกเลิกทั้งหมด. โหมดนี้สามารถระบุได้โดยการส่งตัวเลือก -1 หรือ --single-transaction ไปยัง psql. เมื่อใช้โหมดนี้ โปรดทราบว่าแม้แต่ข้อผิดพลาดเล็กน้อย ก็อาจทำให้การกู้คืนที่ดำเนินการมาหลายชั่วโมงแล้ว ถูกยกเลิกได้. อย่างไรก็ตาม, อาจยังดีกว่าการล้างฐานข้อมูลที่ซับซ้อนด้วยตนเอง หลังจากกู้คืนข้อมูลได้เพียงบางส่วน.
